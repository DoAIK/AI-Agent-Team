$ tsugiai diagnose --sector=manufacturing

$ analyze --type=checklist

// 毎日同じ順序、同じ質問。人は慣れる。慣れは事故を呼ぶ

$ analyze --type=documentation

3つのAIエージェントが、引き継ぎを再構築する

$ tsugiai agents --list

⚡

Template Agent

業種・職種からチェックリストテンプレートを自動生成。2~3の質問で業種特定、プリセット提案。

agent = TemplateAgent(
  model="gemini-3-flash",
  tools=[save_template,
         suggest_items]
)
🎯

Checkout Agent

チェックリストを1項目ずつAI対話形式で確認。テキスト/音声選択。危険キーワード自動検出→深掘り。

agent = CheckoutAgent(
  model="gemini-3-flash",
  tools=[detect_risk,
         deep_dive]
)
📋

Handover Agent

対話結果から引き継ぎノート+アクションアイテムを自動生成。セクション自動分類。

agent = HandoverAgent(
  model="gemini-3-flash",
  tools=[gen_note,
         classify_section]
)

Phase 1 → 2 → 3 → 4

$ tsugiai checkout --flow

PHASE_01

Form Input

フォーム入力。Fisher-Yatesシャッフルでランダム順序表示。パターン化を防止。

$ shuffle(items)
PHASE_02

Photo & AI Judge

写真撮影とAI判定。現場状況を視覚的に記録・検証。

$ judge(photo)
PHASE_03

AI Dialogue

テキスト or Gemini Live音声でAI対話レビュー。危険キーワード自動深掘り。

PHASE_04

Complete

引き継ぎノート+アクションアイテム自動生成。確認・完了。

$ generate(note)

Engineered to Prevent Human Error

$ tsugiai features --unique

TRAP_QUESTION

NGが正の回答

否定形質問でOKを押すとNG扱い。惰性回答を検出して事故を防ぐ。

if answer == "OK" && q.trap:
  result = NG // gotcha
RANDOMIZE

ランダム順序表示

Fisher-Yatesシャッフルで毎回異なる表示順。パターン化を物理的に防止。

for i in range(n-1, 0, -1):
  j = random(0, i)
  swap(items[i], items[j])
RISK_DETECT

危険キーワード自動検出

「火災」「故障」「漏れ」等を検出。自動で2~3回の深掘り質問を生成。

keywords = ["火災", "故障", "漏れ"]
if match: deep_dive(n=3)
VOICE_AI

Gemini Live 音声対話

STT/LLM/TTS全てGemini内部ネイティブ処理。バージイン対応。ハンズフリー。

ws = connect("gemini.live")
ws.send(audio_chunk)
GATE_SYSTEM

ハード/ソフトゲート

必須項目未回答はハードブロック。NG理由確認は提案型。強制しない設計。

HARD: required → block
SOFT: ng_reason → suggest
MULTI_TENANT

マルチテナント

Cloud Firestoreでテナント分離。組織ごとに独立したデータ空間を提供。

db.collection("tenants")
  .doc(tenant_id)
  .collection("checklists")
